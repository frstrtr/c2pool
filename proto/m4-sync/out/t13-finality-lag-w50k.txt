FINDINGS
--------
1. SAFETY: a checkpoint committed at lag L < reorg-depth d is committed over an
   orphan-able tip; after the reorg its committed root is non-canonical and a
   trustless cold-start anchoring to it lands on a DEAD branch. The T9 serving-
   layer safety proof does NOT cover this -- it assumes the commitment canonical.
2. RULE: commit only FINALIZED prefixes -> L >= D_f (=8). This reuses M1's
   finality gate verbatim: a checkpoint is the OwedSettled/BlockFinalized-side
   read of the same overlay the lane settles on. BlockOrphaned never touches a
   committed checkpoint because the checkpoint trails finality.
3. COST: the lag is an ADDITIVE staleness term, delta tail = (C+L) rounds, NOT a
   new asymptotic -- cold-start stays O(W*(1+f*(C+L))). At f=10%, C=5, L=D_f=8:
   delta = +1.3x the W-leaf floor vs +0.5x for lag-0 -- the safety
   premium is +0.8x egress, bounded and one-time per cold-start.
4. DESIGN FEED: checkpoint = {<=8 forest roots + 1 shard-root} committed at depth
   >= D_f behind the tip. Cadence C bounds delta; finality lag D_f bounds safety;
   the two compose additively. Pick C <= (k-1)/f - D_f to keep total delta <= k*W.
